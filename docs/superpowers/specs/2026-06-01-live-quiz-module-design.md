# 設計:海洋偵查員 · 即時挑戰(Live Quiz 模組)

- 日期:2026-06-01
- 狀態:已通過設計討論,待寫實作計畫
- 對象:台南五年級海洋廢棄物 PBL 課程,第二節「課堂即時互動」活動

## 目標

在現有靜態網站新增一個 `/live/` 模組,讓老師用大螢幕投影開房間,學生用平板輸入加入碼後即時搶答 ICC 分類,大螢幕顯示即時排行榜。支援個人 / 小組兩種計分模式切換。

關鍵限制:**即時模組使用一套全新、與前後測遊戲完全不重疊的圖片題庫**,以免學生「背照片」破壞前後測效度。

## 範圍界定

- 這是**新增的課堂活動**,不取代現有的前後測分類遊戲(`game/`)。
- 第一版題型只做「看圖選 5 大類」。ICC 細項(1–20)題型列為後續可選擴充,不在本次範圍。
- 出題節奏為**老師主控**(按一題出一題),非學生自走。

## 角色與畫面

1. **老師主控台**(大螢幕投影)`/live/host`
   - 開房間 → 顯示加入碼 + QR code
   - 選擇個人 / 小組模式
   - 控制出題:出題 → 結束作答 → 公布正解 → 下一題
   - 即時顯示作答分布與排行榜
2. **學生端**(平板)`/live/`
   - 輸入加入碼
   - 個人模式:取暱稱;小組模式:選組別
   - 等待室 → 看圖搶答(點選 5 大類之一)→ 看到自己對錯
3. **共用後端**:沿用現有 Firebase Firestore,以即時監聽(`onSnapshot`)同步。30 人以內負載無虞。

## 上課流程

1. 老師開房 → 大螢幕出現「加入碼 1234」+ QR code。
2. 學生輸碼加入 → 老師畫面即時看到玩家 / 組別清單長出來(等待室)。
3. 老師選個人或小組模式 → 按開始。
4. 每題:
   - 大螢幕顯示一張海廢照片 + 5 個分類按鈕(用 `items.json` 既有的 5 大類顏色)+ 倒數計時。
   - 學生平板點選作答,寫入 Firestore。
   - 老師畫面顯示即時作答人數。
5. 老師按「結束作答」→ 大螢幕公布正解 + 各選項分布長條圖 + 更新排行榜。
   - 計分:答對基礎分 + 速度加分(越快越高,Kahoot 式)。
   - 小組模式:同組成員分數合計;個人模式:各自計分。
6. 跑完所有題 → 顯示最終排行榜。

## 計分規則

- 答對才得分;答錯 0 分。
- 基礎分(固定)+ 速度加分(依答題時間遞減)。
- 小組模式下,小組總分 = 該組所有成員當題得分加總。
- 模式於開房時選定,不在遊戲中途切換。

## 資料模型(Firestore)

與 `game_records` 完全分離,確保前後測數據效度不受影響。

- `live_rooms/{roomId}`
  - `pin`:加入碼(數字)
  - `hostId`:主持者
  - `mode`:`'individual'` | `'group'`
  - `state`:`'lobby'` | `'question'` | `'reveal'` | `'ended'`
  - `currentQuestionIndex`:目前第幾題
  - `questionSet`:本場題目(來自 live 題庫的抽樣)
  - `createdAt`
- `live_rooms/{roomId}/players/{playerId}`
  - `name`:暱稱
  - `group`:組別(小組模式)
  - `score`:累計分數
  - `joinedAt`
- `live_rooms/{roomId}/answers/{questionIndex}_{playerId}`
  - `choice`:所選分類
  - `correct`:對錯
  - `timeMs`:答題耗時
  - `awardedPoints`:該題得分

## 題庫(圖片不重疊)

- 新檔 `data/items.live.json`,結構沿用現有 `items.json`(`category` + `icc_item` + 授權出處欄位)。
- 使用 `scripts/` 既有的 Wikimedia Commons 爬圖腳本,抓一批全新 CC 授權海廢照片,自動套對應 ICC 分類。
- 與 `items.json` 現有 51 張圖**零重疊**。圖片存於獨立資料夾(例如 `images/live/`)。
- 出題沿用既有的分層抽樣精神:每場各分類均衡出題。

## 安全規則

- 學生不需登入即可加入房間,因此需新增一條 Firestore 規則:允許**限定房間範圍的匿名寫入**(只能寫入自己加入的那一間房的 players / answers)。
- 不更動現有 `game_records` 的規則,確保前後測與教師後台不受影響。
- 房間讀取(大螢幕)與寫入(學生)的權限分開設計;考慮以加入碼作為房間範圍的軟性憑證。

## 重用的現有資產

- 5 大類標籤與顏色:`data/items.json` 的 `categories`。
- 視覺風格:`css/base.css` 的設計 token 與既有元件。
- 既有導覽 / 版面樣式。
- Firebase 設定:`js/firebase-config.js`(已移除 IndexedDB persistence)。
